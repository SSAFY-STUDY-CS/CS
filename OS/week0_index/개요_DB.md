# CS 스터디(DB) 1주차 — 개요 & 운영 방식 (8주 커리큘럼)

## 1. 목적(왜 하는가)

- 직무 면접 중 DB 주제 대응
- CRUD 문법을 다시 배우는 것이 아니라 면접에서 자주 나오는 **설계(모델링) / 성능(쿼리·인덱스·실행계획) / 정합성(트랜잭션·동시성) / 운영(장애·복구) / 보안**을 중심으로 이해한다.

> 전제: 기본적인 CRUD/SQL 문법은 알고 있다고 가정한다.
> 
> 
> 대신 기초(JOIN+WHERE, NULL, 실행 흐름, 트랜잭션 경계)는 필요할 때만 짧게 보강한다.
>

## 2. 진행 방식(어떻게 할 것인가)

### (A) 사전 문서 업로드(교과서/개념)

- 각 주차 담당자는 개념 정리 문서(md)를 사전에 업로드한다.
- 문서는 교과서적 정리(정의/그림/예시)가 목적이며, 발표 자료가 아니라 **이해를 맞추기 위한 베이스**다.
- 딥다이브 키워드(참고할 자료/서적 챕터 키워드)

---

### (B) 오프라인 모임: 20% 설명 + 80% 면접 대비

- 오프라인에서는 발표가 목적이 아니라 **면접 대비 시간으로 생각한다.**
- 예상 질문(핵심 5~7개)을 던짐
- 꼬리 질문 1~2개

---

1. **두괄식 1문장(결론)**
2. **왜 중요한가(근거 1~2줄)**
3. **예시 1개(그림/쿼리/상황)**
4. **`트레이드오프` 1개**
5. **함정/꼬리 질문 대비 2개**

> 목표: **정의 암기**가 아니라 왜/언제/대가(`trade-off`)를 말할 수 있어야 한다.

## 5. 딥다이브(선택) — 네카라쿠배/IT 서비스 대비

IT 서비스(대규모 트래픽) 쪽이면 살펴봐도 좋을 것 리스트

1. **대규모 트래픽에서의 읽기 확장**
    - Read Replica / Read-Write Split / Lag 이슈 / 읽기 일관성 전략
2. **캐시 전략(서비스 필수 패턴)**
    - Cache Aside / Write Through / 캐시 무효화 / TTL / Stampede(동시 만료)
    - “캐시 붙이면 생기는 정합성 문제” 답변 프레임
3. **분산/확장 설계**
    - Sharding(수평 분할) / Partitioning / 핫키(Hot Key) / 리밸런싱
4. **고가용성/복구**
    - 장애 전환(Failover) / 백업·복구(RPO/RTO) / PITR 개념
5. **일관성 모델(서비스 의사결정)**
    - 강한 일관성 vs 최종 일관성 / CAP·PACELC(키워드 수준)
    - “어떤 상황에 어떤 선택?” 시나리오
6. **운영 이슈 기반 DB 면접 질문**
    - 슬로우 쿼리 대응(로그→플랜→인덱스/쿼리)
    - 데드락 대응(재시도/락 순서/격리 수준/쿼리 패턴)
7. **보안**
    - SQL Injection 방어(Prepared Statement, 권한 분리)
    - 최소 권한, 감사 로그(키워드)
8. 스캔도 해야겠다.
