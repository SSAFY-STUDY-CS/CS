# [보충5.1] Software Solution 동작 과정

## 1. 초기 시도의 한계와 실패 원인 분석

완벽한 알고리즘이 나오기 전, 단순한 논리들이 왜 상호 배제 조건을 만족하지 못했는지 분석합니다.

### 1.1 version1: Turn 변수 (Strict Alternation)

- **로직:** `while (turn != me);` (내 차례가 아니면 대기)
- **문제점: Progress(진행) 조건 위배**
- **원인**
    - P0이 critical section에 진입하지 않는 경우
    - P0이 진입 후 turn을 초기화 하기 전에 종료되는 경우
    - P0이 두번 연속 진입 불가

### 1.2 version2: Flag 변수 (Check then Set)

- **로직:** `while (flag[you]); flag[me] = true;` (상대가 없으면 내 깃발 들고 진입)
- **문제점:** **Mutual Exclusion(상호 배제) 조건 위배**
- **원인:** P0이 상대 깃발이 내려간 것을 확인한 직후, 깃발을 들기 전에 CPU를 뺏겨 P1이 수행된다면 두 프로세스 모두 임계 영역에 진입하게 됩니다.

### 1.3 version3: Flag 변수 (Set then Check)

- **로직:** `flag[me] = true; while (flag[you]);` (내 깃발 먼저 들고 상대 확인)
- **문제점:** **Progress 및 Bounded Waiting 위배**
- **원인:** 두 프로세스가 동시에 깃발을 들면, 서로 상대방이 깃발을 내리기만을 기다리며 아무도 진입하지 못하는 **Livelock** 상태에 빠집니다.

## 2. Dekker's Algorithm 분석 및 시뮬레이션

두 프로세스(P0, P1) 간의 상호 배제를 보장하는 최초의 올바른 알고리즘입니다. `flag`와 `turn`을 결합하여 '의사 표시'와 '우선순위 양보'를 동시에 관리합니다.

![image.png](image%208.png)

### 2.1 알고리즘 원리

1. 자신의 `flag`를 들어 진입 의사를 밝힙니다. (flag[0] ← true)
2. 상대방도 `flag`를 들었는지 확인합니다. (flag[1] = false)
3.  충돌 발생 시, `turn`을 확인하여 내 차례가 아니면 `flag`를 잠시 내리고 기다립니다. 
(if turn = 1, flag[0] = false)
4. 내 차례가 되면 다시 `flag`를 들고 진입합니다.

### 2.2 시뮬레이션: P0과 P1이 동시에 진입할 때

- **Step 1:** P0과 P1이 동시에 `flag[0]=true`, `flag[1]=true`를 수행합니다.
- **Step 2:** 둘 다 `while(flag[상대])`에 걸립니다.
- **Step 3:** 이때 `turn`이 1이라면, P0은 자기 차례가 아님을 인지하고 `flag[0]=false`로 내립니다. 
(P1에게 길을 열어줌)
- **Step 4:** P1은 `flag[0]`이 내려간 것을 확인하고 임계 영역에 진입합니다.
- **Step 5:** P1이 나올 때 `turn=0`으로 바꿔주면, 대기하던 P0이 다시 `flag[0]=true`를 하고 진입합니다.

## 4. Dijkstra Algorithm 분석 및 시뮬레이션

데커나 피터슨 알고리즘이 2개의 프로세스만 제어할 수 있었다면, 다익스트라는 이를 **N개의 프로세스**로 확장했습니다.

![image.png](image%2010.png)

### 4.1 핵심 개념: flag의 3단계 상태

다익스트라는 프로세스의 상태를 더 세분화하여 관리합니다.

- **IDLE (0):** 임계 영역 진입 의사 없음.
- **WANT_IN (1):** 1단계 진입 시도 (깃발 들기).
- **IN_CS (2):** 2단계 진입 완료 (임계 영역 안 혹은 직전).

### 4.2 시뮬레이션: N개의 프로세스 중 승자 뽑기

1. **1단계 진입:** 진입을 원하는 프로세스들이 자신의 `flag`를 `WANT_IN`으로 바꿉니다.
2. **순번 확인:** `turn`이 가리키는 프로세스가 내가 아니라면, `turn`이 가리키는 프로세스가 `IDLE`인지 계속 확인하며 내 차례를 기다립니다.
3. **2단계 검사:** `turn`을 내 번호로 바꾸는 데 성공했다면, `flag`를 `IN_CS`로 바꿉니다. 그 후 **나를 제외한 다른 모든 프로세스 중에 `IN_CS`인 놈이 있는지 다시 한 번 검사**합니다.
4. **최종 진입:** 아무도 없다면 내가 승자가 되어 임계 영역에 진입합니다. 만약 있다면 다시 1단계로 돌아가 양보합니다.
